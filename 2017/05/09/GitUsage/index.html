Git使用备忘录

May 9, 2017 | 技术

使用

获取项目

git clone url
把服务器端的项目克隆到本地

添加全部

git add .
修改项目后直接添加所有

提交修改

git commit -m “XXX”
提交一次修改记录，注意这只是提交到本地，并不是直接提交到服务器上

获取远程项目

git fetch origin branchName 获取分支更新内容
branchName:分支名

git fetch 获取所有分支所有更新内容

合并代码

git merge origin/branchName

获取远程分支并检出到本地

git checkout -b local_branchName origin/remote_branchName
local_branchName:本地分支名
remot_branchName:远程分支名

创建本地分支

git branch branchName
branchName:分支名

强制还原

git reset –hard

会还原到上个commit，当然不一定跟–hard，我个人比较习惯用hard，因为改的时候有时候会改的很乱，所以就直接会到上个commit，整理思路重新来过

推送至服务器

git push -u origin branchName 把本地分支推送到远程分支
branchName:分支名
origin:主机名

其实如果有多个远程仓库的话应该改一下主机名就行了

ssh代理

一般当项目比较大，用http clone不下来或者安全等级比较高就会用到ssh代理。

  1. 生成公钥
    ssh-keygen -t rsa -C “loginName”
    如果不需要密码的话，连续3个回车。需要密码没试过ヽ(￣д￣;)ノ
    最后得到了两个文件：id_rsa和id_rsa.pub。

  2. 将公钥托管到agent
    $ ssh-add ~/.ssh/id_rsa

  3. 在服务器上添加公钥
    把id_rsa.pub文件里的内容复制进去

  4. 重置连接
    把项目通过ssh代理从服务器上clone下来后，因为不是特别会使用ssh代理，对ssh的权限也只有读的权限，所以想切回http代理，那么怎么弄呢
    git remote set-url origin url
    origin:主机
    url:新的连接
    然后就能愉快的push了

添加额外仓库

git remote add originName url
originName为额外仓库的名称，一般原来的都是origin，所以额外的起一个不一样的已做区分
添加完后使用fetch命令，就能看到有另外一个仓库了
